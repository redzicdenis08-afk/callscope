# Distribution autopilot

This repo is designed so it can keep proving itself without daily babysitting.

## Automated forever

- CI runs on every push and pull request.
- Launch Ready runs on push, pull request, manual trigger, and a weekly schedule.
- The benchmark must keep passing.
- The launch audit checks README, docs, benchmark size, and demo assets.

## Manual once

These parts require account access and should be done once from the relevant UI:

1. Add repository topics from `docs/GITHUB_SETUP.md`.
2. Upload the social preview image.
3. Pin `callscope` first on the profile.
4. Share one clean launch message per channel.

## Launch channels

- Show HN: focus on the tiny benchmark and zero-dependency CLI.
- Reddit: focus on the practical AI voice QA use case.
- LinkedIn: focus on turning call transcripts into decision metrics.
- X: short demo clip plus repo link.

## Rule

One strong launch beats ten weak blasts.
